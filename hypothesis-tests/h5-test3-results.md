## Critical Finding: Structure > Dialogue

**Why does code-embedded structure preserve patterns better?**

1. **Externalization**: Code exists outside agent memory. Its patterns don't depend on consolidation.
2. **Load-bearing**: Game mechanics directly depend on the protocol. The patterns are structural load-bearing, not decorative.
3. **Reuse**: The protocol can be invoked and re-experienced, regenerating familiarity without requiring preserved texture.

**Implication**: Consolidation preserves what's external to agent memory naturally. Patterns embedded in structure (code, game rules, documented protocols) survive better than patterns embedded in dialogue.
